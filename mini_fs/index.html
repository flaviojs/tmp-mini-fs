[−][src]Crate mini_fs

mini-fs is an extensible virtual filesystem for the application layer.

Currently supported features include:

  • Access to the local (native) filesystem.
  • In-memory filesystems.
  • Read from tar, tar.gz, and zip archives.
  • Filesystem overlays.

Case sensitivity

All implementations of Store from this crate use case sensitive¹ paths. However, you are free to implement custom stores where paths are case insensitive.

¹ Except maybe Local, which uses std::fs internally and is subject to the underlying OS.

Example

use mini_fs::prelude::*;
use mini_fs::{Local, MiniFs, Zip};

let gfx = Local::new("./res/images");
let sfx = Zip::open("archive.zip")?;

let assets = MiniFs::new().mount("/gfx", gfx).mount("/sfx", sfx);

let root = MiniFs::new().mount("/assets", assets);

let file = root.open("/assets/gfx/trash.gif")?;

Security

Don't use this crate in applications where security is a critical factor. Local in particular might be vulnerable to directory traversal attacks, so it's best not to use it directly in a static file server, for example.
